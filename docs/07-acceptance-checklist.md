# docs/07 · 总体验收清单

- **文档目的**：提供可勾选的总体验收标准，覆盖功能、并发、权限、实时、一致性、前端、报表、扩展、部署、文档。
- **适用范围**：MVP 及 MVP+ 验收。
- **读者对象**：测试/答辩/Agent。
- **相关文件**：[docs/01](01-requirements.md)、[server/12](../server/12-server-test-and-acceptance.md)、[client/09](../client/09-client-test-and-acceptance.md)。

## 关键结论
- 验收以**并发正确性**与**实时一致性**为红线，任一不过即 MVP 不达标。

## 一、功能验收
- [ ] 学生可登录并按校区/楼栋/楼层/自习室筛选。
- [ ] 学生可查看空位、选时间片、网格选座、提交预约。
- [ ] 管理员可维护校区/楼栋/楼层/自习室/开放时间/座位排布。
- [ ] 座位可启用/禁用，DISABLED 不可预约。
- [ ] 学生可签到、签退、主动取消、查看本人预约记录。
- [ ] 使用中的座位在签退或预约结束时间到达后释放为 FREE（不会永久停留在 USING）。

## 二、并发验收（红线）
- [ ] 10 请求同座同片并发，仅 1 成功，其余 `SEAT_ALREADY_RESERVED`。
- [ ] 断开 Redis 后并发仍不双占（唯一索引兜底生效）。
- [ ] 同一学生超单日次数 → `DAILY_LIMIT_EXCEEDED`。
- [ ] 时间范围非法 → `INVALID_TIME_RANGE`。

## 三、权限验收
- [ ] 未登录访问受保护接口 → `AUTH_REQUIRED`。
- [ ] 学生访问管理端接口 → `PERMISSION_DENIED`。
- [ ] 学生只能操作本人预约。
- [ ] 黑名单用户预约 → `USER_IN_BLACKLIST`，但可登录/查看历史。

## 四、实时推送验收（红线）
- [ ] 看板首次加载获得 `board_snapshot`。
- [ ] 一端预约/释放/签到，另一端秒级收到对应事件并局部更新。
- [ ] SSE 断线可自动重连并重取快照。
- [ ] 有心跳事件保活。

## 五、数据一致性验收
- [ ] 预约成功后 `reservation` 与 `reservation_slot` 一致，Redis 状态与库一致。
- [ ] 超时释放后座位复位为 FREE，slot 释放，看板同步。
- [ ] 取消后状态与座位一致。
- [ ] Redis 与 MySQL 不一致时以 MySQL 为准（可回源重建缓存）。

## 六、前端交互验收
- [ ] 不能选择过去时间/已占座位（前端拦截 + 后端兜底）。
- [ ] 并发失败有友好提示并刷新座位状态。
- [ ] 401 跳登录、403 提示无权限、业务错误码有可读文案。
- [ ] 座位状态颜色语义正确，本人预约高亮。

## 七、报表验收
- [ ] 日均使用率、热门时段、取消率、爽约率、利用率排行可展示。
- [ ] 报表读 `room_daily_stats` 聚合表，非实时全表扫描。
- [ ] 支持按校区/楼栋/自习室筛选。

## 八、扩展功能验收（按已实现范围）
- [ ] 积分：正常签退 +2、临近取消 -1、超时 -3，排行榜正确，流水留痕。
- [ ] 附近空位：不返回无空位自习室，排序符合规则，定位失败可回退手动选择，`NO_AVAILABLE_ROOM_NEARBY` 生效。
- [ ] （若实现）通知：站内可展示与已读。

## 九、部署验收
- [ ] `docker compose up` 起 mysql/redis/backend 成功。
- [ ] 初始化 SQL 可导入种子数据。
- [ ] Knife4j `/doc.html` 可访问。
- [ ] 前端可连后端完成核心流程。

## 十、文档验收
- [ ] 字段/接口/规则变更均已同步对应文档。
- [ ] 术语与 GLOSSARY 一致。
- [ ] 每份文档含固定结构与分级标签。

## 实现约束
- 红线项（并发、实时）不通过不得进入下一阶段。

## 验收标准
- MVP：一至七 + 九 + 十全部通过。
- MVP+：追加第八节对应项。
