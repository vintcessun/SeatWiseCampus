## R1-注册入口与验证码（UI 改进）· P2

### 问题描述
1. 注册按钮作为页面右下角角标，位置不显眼、布局拥挤，视觉效果不佳。
2. 缺少图形验证码（CAPTCHA）功能，存在自动化注册风险。

### 待确认/修复项
- [ ] 重新设计注册入口布局（登录页底部链接 / 独立卡片 / 弹窗等）
- [ ] 增加图形验证码（建议集成 EasyCaptcha 或类似方案，后端生成 + Redis 校验）
- [ ] 验证码仅在注册/登录频次异常时触发，避免影响正常用户体验

### 待修复项

## R3-自习室管理功能缺失 · P1

### 问题描述
1. **缺少删除自习室**：管理端没有删除自习室的入口，已废弃/错误录入的自习室无法移除。
2. **楼栋选择固定**：新增自习室时，可选的所属楼栋固定为两种，没有"新增楼栋/校区"的能力。

### 待修复项
- [ ] 管理端自习室列表增加「删除」操作（逻辑删除，需校验是否有未来预约）
- [ ] 基础数据管理增加「新增楼栋」「新增校区」功能
- [ ] 新增自习室对话框的楼栋下拉框从数据库中动态读取（而非硬编码）
- [ ] 考虑在数据库中添加楼栋/校区管理报表

## R4-缺少临时关闭自习室功能 · P1

### 问题描述
- 自习室没有「暂时关闭」的能力。管理员遇到维修、考试占用等场景时，无法临时关闭自习室。
- 关闭后应：禁止新预约、不影响已有预约或提示已预约用户调整。

### 待修复项
- [ ] 自习室状态增加 CLOSED / MAINTENANCE 等状态
- [ ] 管理端自习室详情页增加「关闭/开放」切换按钮
- [ ] 关闭时前端提示可能影响的预约数量
- [ ] 关闭自习室时联动公告功能，通知已预约学生（见 R10）

---

## R5-座位排布仅支持单个过道 · P1

### 问题描述
- 座位排布编辑器的「快速生成网格」只能指定一个过道列（`aisleCol`）。
- 实际教室排布中可能有：多个过道、不同行的列数不同（如后排中间少一列）、柱子、讲台等不规则情况。
- 管理员应能自主设置每个单元格的类型（SEAT / AISLE / EMPTY / DISABLED）。

### 待修复项
- [ ] 排布编辑器中每个单元格支持独立切换 cell_type（SEAT / AISLE / EMPTY / DISABLED）
- [ ] 增加多过道支持（或完全交由手动编辑，不依赖快速生成模板）
- [ ] 支持每行不同列数（不规则房间排布）

## R6-缺少子管理员账号 · P3

### 问题描述
- 当前只有 ADMIN 角色。实际使用中需要多个子管理员（如各楼层管理员），拥有有限权限（仅管理自习室/查看报表，不能管理其他管理员）。
- 缺少"新增子管理员"的入口。

### 待确认/修复项
- [ ] 新增角色 ADMIN_SUB（或类似），权限范围缩小
- [ ] 管理端增加「管理员管理」页面（主管理员可创建/禁用子管理员）
- [ ] 权限矩阵更新：子管理员不可管理其他管理员、不可修改系统配置

---

## R7-管理端时空座位图 + 选座预约结合 · P2

### 问题描述
1. **管理端缺少时空座位图**：管理端查看自习室时，缺少时空占用可视化，无法直观看到某天的整体占用时间分布。
2. **学生端时空座位图与选座脱节**：当前学生端时空座位图仅展示，不能直接在时间轴滑块上选择时段并提交预约——需跳转到选座页再重新操作。

### 待修复项
- [ ] 管理端复用或新增时空座位图组件（可基于 Replay.vue 改造）
- [ ] 学生端时空座位图增加「在此时间范围预约」一键功能，减少跳转步骤
- [ ] 时间轴滑块上直接选择预约起止时间（当前只能选开始时刻）

## R8-缺少预约须知/规则提示 · P2

### 问题描述
- 学生端没有任何关于预约规则的说明。
- 学生可能不知道：签到窗口只有 15 分钟、提前 30 分钟内取消会扣分、连续爽约进入黑名单等规则。
- 可能导致无意识违规，影响使用体验。

### 待修复项
- [ ] 选座页/预约提交页增加「预约须知」折叠面板或弹窗
- [ ] 内容包括：签到窗口说明、扣分规则、黑名单规则、单日预约次数限制
- [ ] 首次预约或积分发生变化时弹窗提示规则

## R9-通知类型显示异常（Bug）· P2

### 问题描述
- 通知中心中，不同类型的通知（公告/提醒/预约提醒等）均显示为 `announcement`，未正确区分通知类型。
- 通知列表缺少类型图标或标签区分，可读性差。

### 待修复项
- [ ] 确认通知 `type` 字段在前端渲染时的映射逻辑
- [ ] 通知列表增加类型图标/标签（如公告=📢、提醒=⏰、积分=⭐）
- [ ] 修正类型显示，确保不同类型通知有明确视觉区分

## R10-修改座位未校验预约 + 缺少关闭联动公告（数据风险）· P1

### 问题描述
1. **修改座位未校验未来预约**：管理员修改/删除存在未来预约的座位时，无任何警告或禁止，可能导致已预约学生到馆发现座位不存在/被移动。
2. **关闭自习室缺少公告联动**：关闭自习室时没有自动公告通知受影响学生，学生可能到馆才发现。

### 待修复项
- [ ] 保存/修改座位排布前，后端校验该自习室是否存在未来（待签到/进行中）的预约
- [ ] 若有未来预约，前端弹窗警告管理员具体影响的预约数量
- [ ] 最佳实践：仅允许在自习室关闭（CLOSED）状态下修改座位（见 R4）
- [ ] 关闭自习室时联动公告功能，自动生成公告通知已预约学生（复用 `AnnouncementService` + `NotificationService` 推送）

---

## R11-附近空位需定位 + 位置管理 · P3

### 问题描述
1. **附近空位功能未完善**：当前 `nearby` 页面功能基础，未结合学生真实定位，无法提供精确的"附近"推荐。
2. **缺少位置管理**：管理员不能管理建筑/自习室的地理位置（经纬度），也没有地图选点功能。位置坐标仅依赖初始数据。

### 待修复项
- [ ] 管理员端增加「位置管理」页面，支持为校区/楼栋/自习室设置经纬度
- [ ] 考虑接入简单地图选点（如高德/百度地图 LBS API 或静态图坐标拾取）
- [ ] 限制合法坐标范围（校园内），防止坐标偏离
- [ ] 学生端 `nearby` 建议结合浏览器 `Geolocation` API 获取当前位置
- [ ] 定位失败时回退手动选择校区/楼栋
