# ROADMAP.md · 开发路线图（P0–P9）

- **文档目的**：把 SeatWise Campus 的开发拆成可独立交付、可验收的阶段，供人与 Agent 按序推进。
- **适用范围**：全项目。
- **读者对象**：开发者、Coding Agent。
- **相关文件**：[docs/04-mvp-scope.md](docs/04-mvp-scope.md)、[server/13-llm-server-implementation-guide.md](server/13-llm-server-implementation-guide.md)、[client/10-llm-client-implementation-guide.md](client/10-llm-client-implementation-guide.md)。

## 关键结论
- **P0–P6 = MVP**，**P7–P8 = MVP+**，**P9 = 后续扩展**。
- 每阶段必须保持系统可运行、可演示。**禁止跨阶段一次性铺开**。
- 每个阶段完成后更新对应设计文档与本路线图勾选状态。

## P0 · 文档与脚手架
- **目标**：完成全部设计文档与 Agent 指令；搭建 client/server 空工程骨架与 Docker Compose。
- **交付物**：本仓库全部 Markdown；`server` Spring Boot 骨架；`client` Vite 骨架；`docker-compose.yml`（mysql/redis/backend）。
- **依赖**：无。
- **验收标准**：文档树完整、可运行 `docker compose up` 启动依赖、前后端空工程可启动。
- **可砍范围**：Nginx 容器可后置到 P6+。

## P1 · 登录与基础数据管理
- **目标**：Sa-Token 登录；管理员维护校区/楼栋/楼层/自习室/开放时间。
- **交付物**：`sys_user/campus/building/study_room` 表与 CRUD 接口；学生/管理员登录页与基础管理页。
- **依赖**：P0。
- **验收**：两类角色登录成功、权限矩阵生效、基础数据可增删改查。
- **可砍**：开放时间可先固定默认值。

## P2 · 座位排布与查询
- **目标**：座位网格建模；管理员编辑座位排布；学生按条件查询自习室与座位。
- **交付物**：`seat` 表 + 布局 JSON；`AdminLayoutEditor`；筛选与座位查询接口。
- **依赖**：P1。
- **验收**：排布可保存/回显；`cell_type=SEAT` 才可预约；查询返回座位静态状态。
- **可砍**：复杂网格工具（复制行/批量禁用）可后置。

## P3 · 核心预约
- **目标**：时间片预约 + 并发控制（Redisson 锁 + 唯一索引兜底）。
- **交付物**：`reservation`/`reservation_slot` 表；预约接口；单日次数限制。
- **依赖**：P2。
- **验收**：10 并发同座同片仅 1 成功；超次返回 `DAILY_LIMIT_EXCEEDED`。
- **可砍**：跨天/连续多时段合并预约可后置。

## P4 · 签到、签退、超时释放、黑名单
- **目标**：签到窗口、签退/自动完成、超时自动释放、爽约计数、黑名单。
- **交付物**：签到/签退/取消接口；Redisson DelayedQueue 超时释放任务 + 自动完成任务；`blacklist_record`。
- **依赖**：P3。
- **验收**：超时 15 分钟未签到自动释放并复位；使用中座位经签退或结束时间自动完成后释放为 FREE；爽约达阈值进黑名单；黑名单用户预约返回 `USER_IN_BLACKLIST`。
- **可砍**：未主动签退的额外扣分可后置（自动完成默认不扣分）。

## P5 · 实时热力图
- **目标**：快照 + SSE 增量推送，多客户端同步。
- **交付物**：SSE 端点；`HeatmapBoard`/`SeatGrid`；断线重连。
- **依赖**：P3、P4。
- **验收**：两客户端同看板，一端预约/释放，另一端秒级更新。
- **可砍**：热力图高级样式（渐变/动画）可后置。

## P6 · 数据报表
- **目标**：使用率、热门时段、取消率、爽约率、利用率排行。
- **交付物**：`room_daily_stats` 聚合表 + 定时任务；报表接口；ECharts 报表页。
- **依赖**：P3、P4。
- **验收**：报表读聚合表；口径与文档一致。
- **可砍**：可先只做“日均使用率 + 利用率排行”。

> —— 以上 P0–P6 构成 MVP ——

## P7 · 积分排名（MVP+）
- **目标**：守约加分/爽约扣分/排行榜。
- **交付物**：`score_record` + `credit_score`；积分结算挂到预约状态机；排行榜接口与页面。
- **依赖**：P4。
- **验收**：正常签退 +2、临近取消 -1、超时未签到 -3；排行榜正确。
- **可砍**：管理端积分规则可配置化可后置（先硬编码规则）。

## P8 · 最近空位推荐（MVP+）
- **目标**：按距离 + 空位 + 开放状态推荐自习室。
- **交付物**：坐标字段；`room_availability_summary`、`nearest-available-room` 接口；附近空位页。
- **依赖**：P2、P3。
- **验收**：不返回无空位自习室；排序符合“同楼栋 > 同校区最近 > 空位更多”。
- **可砍**：浏览器 Geolocation 可后置，先手动选位置。

## P9 · AI 推荐与通知提醒（后续扩展）
- **目标**：AI 错峰/座位推荐、预约/超时/解禁提醒。
- **交付物**：`notification` 表与通知中心页面；AI 推荐接口占位。
- **依赖**：P6、P7、P8。
- **验收**：接口可返回占位/规则结果；通知可在站内展示。
- **可砍**：真实模型与外部推送通道整体可砍，仅保留接口与文档。

## 给 AI Coding Agent 的提示
- 开工先确认当前 P 阶段，只做该阶段交付物。
- 每完成一阶段：更新对应 `server/*`、`client/*` 文档并在本文件记录状态。
- P3/P4/P5 涉及正确性与实时性，务必先读 server/05、server/06、server/07。
